Hexo搭建自己的博客笔记

本文还未完善，谨慎参考，谢谢

Hexo部署

根据Hexo官方文档,需要安装Node.js(且版本不能低于10.13),还有Git。

安装Hexo

可以使用npm安装,终端输入

npm install -g hexo-cli

或者

npm install hexo

安装以后可以使用以下两种方式执行Hexo:

npx hexo <command>

或者添加如下环境变量，可以直接使用Hexo :

echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile

搭建博客基本框架

执行以下命令

hexo init <folder>
cd <folder>
npm install

其中为博客框架放置的位置，可以放置在一个文件夹内，如果没有指定,则会在当前目录下载框架
举个例子，放置在家目录的myblog文件夹中

hexo init $HOME/myblog
cd $HOME/myblog
npm install

如果提示权限不足，使用sudo

sudo hexo init $HONE/myblog
cd $HOME/myblog
sudo npm install

配置HEXO

在框架所在目录（如上$HOME/myblog）中的==_config.yml==, 为Hexo的配置文件，使用编辑器编辑，详情见Hexo官方文档

如果你也使用github作为博客部署的仓库，则需要修改如下配置

# URL
## Set your site url here. For example, if you use GitHub Page, set url as 'https://username.github.io/project'
url: https://<yourname>.github.io
permalink: :year/:month/:day/:title/
permalink_defaults:
pretty_urls:
  trailing_index: true # Set to false to remove trailing 'index.html' from permalinks
  trailing_html: true # Set to false to remove trailing '.html' from permalinks

将修改为你的github用户名

deploy:
  type: git
  repo: <repository url> #https://bitbucket.org/JohnSmith/johnsmith.bitbucket.io
  branch: [branch]

将repo修改为仓库所在链接
将branch修改为仓库的主分支master（某些时候为main，打开自己的仓库查看）

安装hexo-deployer-git

npm install hexo-deployer-git --save

这时可以尝试生成站点文件推送至远程库，执行

hexo clean && hexo generate && hexo deploy

提示输入用户名与密码，用户名为github用户名，密码为github的令牌（某个版本之后不支持密码提交推送）

提交完成后可以在==https://.github.io==,查看当前博客页面

Hexo server

提交更改至github需要谨慎，所以半成品可以使用hexo-server在本地查看博客

安装hexo-server

npm install hexo-server --save

启动hexo-server

hexo server

根据提示在http://localhost:4000（或http://127.0.0.1:4000)，查看制作好的页面
