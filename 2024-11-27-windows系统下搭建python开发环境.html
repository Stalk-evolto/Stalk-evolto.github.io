在我看来，python就是我们与计算机对话的一种方式。
就像我们平时给别人安排工作，在中国你可以用普通话告诉对方要做什么、怎么做…因为我们都能听懂普通话，交流没有障碍。但是今天你需要给一个只会英文的英国技术员安排工作，但是你又不会英语，怎么办？

安装一个适合代码编写的编辑器

从上一个例子，我们使用了记事本来编写代码。在编写少量代码时不会有什么问题，但是现代程序动不动就几百上千行，如果这个时候你再用记事本编辑，效果是这样的：

那么怎么办呢？
我们可以安装一款适合代码编写的编辑器。

适合编写代码的编辑器有很多，例如Visual Studio Code（简称VS Code），Vim，Emacs，Notepad++…

有很多很多，我目前最常用的编辑器是Vim，别的编辑器了解不深。但是考虑到本篇文章受众在Windows环境下，所以学习了一下VS Code的安装配置。虽然我在这只介绍两种编辑器的安装，你也可以凭你个人喜好自行安装一款你喜欢的编辑器使用。

  • Visual Studio Code
  • Vim

注: 为什么没有Pycharm？因为使用集成开发环境（IDE)虽然可以让你快速上手编写Python程序，但是它掩盖了很多其中的细节，例如虚拟环境是如何创建与激活的。而且IDE通常有一定的功能在IDE中需要付费，但是这是可以通过你自己搭建而免费使用的。所以我建议少使用IDE，所以本篇文章也不会介绍它。

Visual Studio Code

  • VS Code的官方网站: https://code.visualstudio.com/

安装

  1. 打开Edge浏览器，在地址栏输入https://code.visualstudio.com/，回车后来到VS Code官网。

    点击箭头指向的Download，来到下载页面。

    VS Code的安装程序有那么多种，我们要选哪一种呢？根据我们之前[安装python解释器](# 安装python解释器)时查看的电脑配置，我们应该选择x64。
    那么user installer与system installer我要选择哪一个呢？这要根据你的电脑使用情况来定，如果你的电脑只有你一个用户，那么选择user installer或者system installer效果是一样的（当你不使用管理员账户Administrator使用VS Code时)；如果你的电脑有多个用户账户，而你只想给自己的账户安装VS Code，那么选择user installer。如果你想所有账户都能使用VS Code，那么选择system installer。
    我在本篇文章里演示user installer安装程序的安装过程（其实system install的安装过程也类似，也就是安装路径不需要更改而已）,点击图片红框处下载VS Code的user installer安装程序。

  2. 开始下载时，会跳转到VS Code官方文档，你在等待下载的过程中可以先浏览一下，了解VS Code的简单使用方法。
    下载完毕后，点击在文件夹中显示

  3. 由于是user installer，直接双击VSCodeUserSetup-x64-1.95.3.exe，开始安装

    弹出安全警告，我暂时不会在windows下验证软件包的GPG签名，不过由于是在官网下载的，所以暂时认为下载的软件包是没被篡改的今后我学到了再修改这里。点击运行

  4. 请认真阅读的License，对我们的使用有一定限制，例如不能逆向VS Code获取其源代码、VS Code会收集用户部分数据、VS Code有出口限制等等。阅读完与自己需要用VS Code要干的事没有冲突的话，点我同意此协议

    如果你没有阅读该协议，直接点我同意的话，还是建议你去官网阅读此协议:

  • https://code.visualstudio.com/License

  1. 这份提示是因为我们使用的是user installer只会为当前用户安装VS Code，管理员账户Administrator无法使用，点击确定即可。

作者的碎碎念

在学习python的过程中你可能会遇到很多困难，例如明明我写的和书上一样，为什么这么多Warning和Error？、我的代码只有120行，为什么报错说1000行错误？、这个报错说缺少Microsoft Visual C++ 14.0，为什么我用Python还要用C++的编译器？等等。

不用惊慌，学习一门新的技能遇到问题是难免的，而且你遇到的问题很多前辈同样遇到过，他们或许已经找到了解决办法，你可以将问题复制到浏览器搜索框搜索一下，或许会有解决办法。或者去官方网站查看文档还有bug汇总，有可能是官方还没解决的问题…

上述这些可能现在的你听起来一头雾水。没关系，学习本就不是一蹴而就，就算看完官方文档也不是“把python学完了”；有的只是一个又一个问题的解决、今天比起昨天的自己有进步。日积月累，再回过头看，你已经比刚开始print("Hello World!!)的自己强大了很多很多。我就这样过来的

在这过程中，你也可能发现python的很多弊端：缩进严格、数据类型不明显、程序打包后占用太大…从而转向其他语言（我除了使用python同时也经常使用C语言），也可能会听到“python是垃圾语言，xx语言才是最好的编程语言”这样的言论。其实每一种编程语言都有自己擅长的地方，没有什么语言最好，有的只是这个工作用这个语言比较适合，这个工作可能需要与另一门语言编写的程序合并。为了虚荣心而拉踩是没有意义的，扬长避短才是我们需要思考的。

那么开启属于你自己的编程之旅吧~~

如果你的Edge浏览器点进去主页是这样：



那说明你的Edge浏览器默认主页被篡改了，在萌新阶段大概率是你的杀毒软件开启了浏览器主页保护功能。下面列举几个常见的杀毒软件关闭主页保护的方法。

  • 360安全卫士
  • 火绒安全
  • 金山毒霸

  1. 打开360安全卫士，看到如下界面：
  2. 点击功能大全，再点击搜索工具，搜索浏览器防护设置,点击后出现如下界面:
  3. 将3个已锁定的选项解锁，即可关闭360对Edge浏览器默认主页的篡改。

火绒安全

  1. 打开火绒安全，看到如下界面:
  2. 点击左下角的设置(红框框住的齿轮),会打开设置窗口:
  3. 点击系统防护，再点击浏览器防护，关闭浏览器防护的开关(红框框住的开关),即可关闭火绒安全对Edge浏览器默认主页的篡改。

金山毒霸

  1. 打开金山毒霸，看到如下界面:
  2. 点击右下角的百宝箱，进入到如下界面:
  3. 搜索浏览器保护，跳转到如下界面:
  4. 将红框框住的开关全部关闭，即可关闭金山毒霸对Edge浏览器主页的篡改。

  4. 关闭掉这两项的开关

添加环境变量

什么是环境变量

举个例子，我现在准备做蛋炒饭。需要用先用电饭煲煮熟米饭，再用炒锅炒鸡蛋，最后将煮熟的米饭与煎蛋混合再加入调味料，就做好一份香喷喷的蛋炒饭啦。

那么我要到哪里找大米、电饭煲、鸡蛋、炒锅和调味料呢？

哦哦，大米在厨房柜子里，电饭煲和调味料在厨房桌子上，炒锅在厨房的燃气灶上，鸡蛋在客厅的冰箱里。

那么我需要到客厅和厨房寻找我做这碗蛋炒饭所需的食材与工具。

现在，将我换成你的电脑，将大米、鸡蛋、调味料看成电脑里需要处理的文件，电饭煲、炒锅看成电脑里的软件、客厅、厨房看成电脑里的文件夹。

环境变量就是你告诉电脑：我需要运行这个软件，你需要去C盘的AAA文件夹、D盘的BBB文件夹、D盘的EEE文件夹找找这个软件在不在，如果在就运行它，如果不在告诉我没找到。

显然环境变量就是电脑找工具的目录的集合

添加环境变量

那么，我们要怎么添加环境变量呢？在Windows可以通过GUI界面设置环境变量。

  1. 进入设置，点击系统-系统信息。

  2. 点击高级系统设置

  3. 在弹出的窗口里，点击环境变量

  4. 在新弹出的窗口里可以看到系统环境变量和用户环境变量
    在这我们修改用户环境变量的Path（如果你的电脑一般只有你一个用户使用python）

  5. 还记得python的安装路径吗？如果忘记了，就按一下win键，点击图示搜索框

    在搜索框中搜索python,可以看到以下界面

    点击Python 3.13 (64bit)的打开文件位置

  6. 弹出Python 3.13 (64bit)快捷方式所在的目录

    右键    Python 3.13 (64bit)快捷方式，点击属性

    这里的目标就能看到python 3.13的安装目录了，但是为了让你更直观的了解路径，我们点击打开文件所在位置

  7. 弹出python 3.13安装目录

    点击红框位置，会显示这个目录的绝对路径

    将这段路径复制下来

  8. 回到环境变量编辑的界面，点击新建，将刚刚复制的路径粘贴到新出现的行里

    然后在路径末尾添加一个反斜杠\（在Enter键的上方）

  9. 回到python的安装目录，我们还需要将该目录下Scripts目录添加到环境变量中
    双击Scripts，进入该目录

    点击红框位置

    将出现的绝对路径复制下来

  10. 回到环境变量编辑界面，点击新建，将刚刚复制的路径粘贴到新出现的行里

    然后在路径末尾添加一个反斜杠\（在Enter键的上方）

  11. 添加完毕后，点击一下刚刚添加的环境变量，再点击上移，使刚刚添加的环境变量都处于靠前位置。

    点击确定退出环境变量编辑

12.测试一下环境变量是否添加成功
打开一个终端，输入python回车,成功进入python shell即为成功

首先我们按照正常配置的过程开始配置。

  1. 打开VS Code

    点击左下角的设置（齿轮图标）

    点击配置文件

    进入配置文件编辑页面

  2. 点击新建配置文件

    将该配置文件的名称，改为Python，再点开复制自

    不对劲，官网不是这样的呀！！官网给的图片应该是这样

    在官网的方案中选择Python配置文件模板后，点击创建即可创建配置文件

经过分析，适应为配置文件模板是存放在微软的服务器上的，每次使用都需要连接服务器加载，而国内网络连接微软的服务器费点劲。如果你会使用某些网络工具能脸上微软服务器，那么请尝试开启你的工具再重新进行上述操作即可正常配置。

如果你没有这门手艺，那么我导出了一份配置文件，是用官方的配置文件模板配置的，你可以下载下来

  • Python.code-profile源代码

在弹出来的选项,点击选择文件

查看配置无误后，点击创建
